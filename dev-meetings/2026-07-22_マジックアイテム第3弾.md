## 第1ラウンド：ブレスト（各専門領域から方向性を1つ）

【ガイウス】さて、第3弾だ。第1弾が「対竜」、第2弾が「対アンデッド」——どちらも**敵から逆算した装備群**だった。構造としては美しいが、そろそろ限界が見える。次に「対オーク」を作っても、プレイヤーは「またか」と思うだけだ。だから俺が出すのは**入手の物語**の方向。闇市ポドルプラザに「ワケあり棚」を作る。強いが代償のある品を、自己責任で買う。Gold Box にも呪われた剣はあったし、ゲームブックの「その剣を手に取るか？」の緊張はこのゲームにまだ無い。

【レンツ】俺は真逆に、**穴を埋める**方を出す。棚卸しの通り、gated 武器は3本とも戦士の片手剣で、`rollMagicItemDrop` は非戦士リーダーだと武器候補を1本も積まない（index.html:17030）。つまり**盗賊や魔法使いでプレイしている人は、マジック武器という体験そのものに触れられない**。5職に一点物を1本ずつ足す。効果は既存フィールドの組み合わせで足りる——lifeStealDice も critStun も shockwave も、もう動いてる。データ追加が主体だ。

【ミサキ】私は**防具と盾**。gated がゼロっていま初めて聞いたけど、それ、もったいなさすぎる。武器は光るのに、着てるものは最後まで「チェインメイル」って画面で何も起きないんですよ。「聖別された剣」を実装した時に金色のVFXが完全配線済みで眠ってた、みたいな話があったでしょ。同じことが防具でも起きてる。**装備した瞬間に画面が変わる鎧**。被弾したときに一瞬だけ膜が張るとか。

【ノエル】私はもっと手前。**戦闘中に何もできない**のが引っかかってる。オートバトルだから当然なんだけど、HPが減っていくのを見てるだけの時間が長い。だから**出発前に持ち込む使い切りの品**——HPが一定を切ったら自動で飲む治癒の薬みたいなの。プレイヤーの判断は「何個持っていくか」に集約される。これなら片手・電車の中でも成立するし、準備画面に楽しみが増える。

## 第2ラウンド：絞り込み（制約で間引く）

【ノエル】ガイウスさんの呪いの品、正直きついと思う。**オートバトルで呪われたら、プレイヤーは何もできない**んですよ。TRPGなら「街に戻って解呪してもらう」って動けるけど、うちは出発したら最後まで自動。「装備が外せません」って言われても、画面の前でただ負けるのを見るだけ。それ、面白い緊張じゃなくてただのストレスです。

【ガイウス】……一理ある。ならば呪いを「戦闘中の枷」ではなく「**恒常的なトレードオフ**」にすればいい。攻撃力は上がるが最大HPが減る、命中は上がるがACが下がる。呪いというより**諸刃**だ。装備するかは酒場で決められるから、プレイヤーの判断は保たれる。

【レンツ】その形なら実装は現実的だ。enhancement のマイナス値と hpBonus のマイナスで済む。……ただ、慶彦さんの制約を思い出してほしい。**「出荷時に消す物を増やさない」**。闇市に新しい棚とUIを生やすと、dev shop とは別の陳列導線がもう1本増える。いま gated 武器・防具・盾は**足すだけで武器屋に1G自動陳列される**（`DF_DEV_MAGIC_SHOP`）から、テストに追加コードが要らない。ここは大きい差だ。

【ノエル】同じ理由で、私の消耗品案も自分で叩いておきます。**在庫の持ち込みUIが完全新規**。所持数の永続化、準備画面の増設、index と tavern の二重同期。ぜんぶ新規で、しかも dev shop の1G陳列に乗らない。「1個だけ試す」ができない。楽しさは信じてるけど、第3弾の最初の一手としては重いです。

【ミサキ】防具はどうですか。gated を1つ足すだけなら、武器屋に1Gで並ぶんですよね？

【レンツ】並ぶ。`renderShop` の gated skip がフラグで外れるから、**防具も盾も無改修で陳列される**。そこは効く。問題は効果の方で、防具は現状 blockChance と knockbackReduce と enhancement しか持ってない。「被弾時に膜が張る」なら発火点は既存の被ダメ経路に挿すだけだが、**常時オーラは別**だ。

【ノエル】常時オーラは止めます。カメラ移動のガクガクを16.9ms/frameまで削ったばかりでしょう。パーティ全員が常時発光したら簡単に戻る。**演出は「被弾の瞬間だけ」に閉じるなら賛成**。

【ミサキ】……のむ。むしろ被弾の瞬間だけの方が「効いてる」のが伝わるから、そっちの方が正しいかも。常時光ってると慣れて見えなくなるし。

【ガイウス】レンツの5職案について1つ言わせてくれ。**プールが希釈される**問題はどうする。いま抽選プールは gated 武器と gated アクセを混ぜて一様抽選だ。5本足したら、既存の対竜・対アンデッド装備が出る確率が目に見えて落ちる。

【レンツ】そこは設計で潰せる。**武器は「リーダーの職業のプールからのみ」積む**——いまの warrior ガードを「leaderClassKey のプール」に一般化するだけだ。そうすれば1周のプールサイズはほぼ変わらない。むしろ非戦士でプレイした時に初めて武器がプールに乗るから、**体験としては増えて、確率としては薄まらない**。

【ノエル】それは筋がいい。……で、5職ぶん作るなら、**どの職から作っても1本で成立する**設計にしてください。「5本揃わないと意味がない」だと1周のテストが重い。

【ガイウス】了解した。では候補を4つに整理しよう。

- **ざっくり規模**: 中（1職ずつ刻める。1本でも成立する）
- **既存流用度**: **高**（lifeStealDice/critStun/shockwave/bonusDmgType すべて実装済。dev shop も無改修で1G陳列）

- **ざっくり規模**: 中

- **推し**: ガイウス ／ **慎重**: レンツ（マイナス値がどこで壊れるか未知数。闇市に棚を作る版は「出荷時に消す物が増える」ので不採用）

# 開発会議〔第2段：実装計画〕：五職の一点物 — クラス別マジック武器

**選択案**: dwarf/cleric/mage/elf/rogue の武器プール末尾に gated マジック武器を1本ずつ追加し、`rollMagicItemDrop` の warrior ガードを「リーダーの職業プール」へ一般化する。
**ゴール**: この案を実装に落とす開発計画書をつくる

---

## 第1ラウンド：設計

【レンツ】まず良い知らせから。**主人公側はほぼ無改修で効く**。`getLeaderWeapons()`（index.html:9669）が既にリーダーの職業を見て `ALLY_EQUIP_POOLS[leaderClassKey].weapons` を返しているし、`getCurrentWeapon()` がそれを引いて `applyWeaponSpecialEffects(curWeapon, …)`（index.html:16840）へそのまま渡る。つまり**盗賊の武器に `bonusDmgDice` を書けば、盗賊リーダーで撃った時に既存の属性エフェクトがそのまま乗る**。仲間側も `applyAllyEquipment`（16131-）が `bonusDmgDice/Type/vsUndead` を機械的にコピーしている。**新規VFXはゼロ**だ。

【ミサキ】え、じゃあ絵の仕事が無い？

【レンツ】そう。属性は fire / cold / lightning / holy の4つが完全配線済み（`ELEM_NAME`, index.html:16538）。既存の色を選ぶだけだ。……で、悪い知らせがある。**装備の所持が「共有ティア」なんだ**。

【ノエル】共有？

【レンツ】`ownedEquip` は `{weapons:[0,1,2,…]}` という**インデックスの配列**で、職業を持たない（tavern.html:2977-3025）。`shopPool()` はリーダーの職業プールを引く（tavern.html:3791）。つまり「tier 2 を所持」は「全職の2番目の武器を所持」を意味する。これは通常装備では**意図した設計**だ——ロングソード+1 を買えばドワーフも +1 の斧を持てる。

【ガイウス】……待て。ということは、戦士でフロストブランドを引いた後に盗賊でプレイしたら？

【レンツ】**盗賊の新しい魔剣がタダで手に入る**。フロストブランドは warrior の tier 4、新しい盗賊の剣も append すれば tier 4 だからな。逆方向も起きる。

【ガイウス】それは駄目だ。**1回のドロップで5職ぶんの魔剣が付いてくるなら、レアである意味が消える**。「マジックアイテムは超低確率、何が出るか楽しみ」という前提そのものが壊れる。

【ノエル】しかも一番たちが悪いのは、**開発中は絶対に気づけない**ことです。`DF_DEV_MAGIC_SHOP` が有効な間は全部1Gで並ぶんだから、「あ、もう持ってる」って思っても誰も疑わない。**出荷でフラグを false にした瞬間に初めて表に出るバグ**ですよ、それ。

【レンツ】……その通りだ。過去に同じ形のバグを一度踏んでる。`loadOwnedEquip` の高水位再構成が gated を芋づるで所持化する抜け穴を `dfWeaponTierGated`/`dfOwnedInitRange` で塞いだやつだ。しかもその `dfWeaponTierGated(t)` は **`WEAPONS[t]`（戦士配列）決め打ち**（tavern.html:2982）。非戦士のプールを見ていない。今回の追加で、この決め打ちが本格的に嘘になる。

## 第2ラウンド：肉付けと衝突

【ガイウス】ならば所持の持ち方を変えるべきだ。gated だけは「何番目か」ではなく「**どれを持っているか**」で管理しろ。指輪が `accessoryIds` で id 管理されているのと同じ形だ。前例はもう館の中にある。

【レンツ】賛成だが、工数を正直に言う。触るのは4箇所だ。①`dfWeaponTierGated` をクラス対応に、②`dfIsOwnedTier`/`dfAddOwnedTier` の weapon 分岐で gated なら名前リスト（`dragonfighters.ownedGatedWeapons`）を見る、③`allyEquip` ロードのクランプ `Math.min(…, inventory.weaponIdx)`（tavern.html:3398）が gated 所持を潰すので緩める、④index 側で `rollMagicItemDrop` が非戦士リーダー時に `dragonfighters.allyEquip[classKey].weapon` と名前リストの両方へ書く。……**小さくはない。でも1回やれば5職ぶん効く**。

【ノエル】そこは払うべきコストだと思います。逆に聞きますけど、**この基盤を入れずに1本だけ先に出す**のは駄目ですか？

【レンツ】駄目だ。1本目から漏れる。**基盤と1本目は同じフェーズで出すしかない**。

【ミサキ】絵の仕事が無いなら、私は**5本の「性格」**を詰めます。属性が4つしかないのに5本作ると、必ずどこかで被るんですよ。だから属性だけで差をつけるのは無理。**効果の組み合わせで性格を出す**べきです。ドワーフは範囲を焼く、僧侶は殴って自分が回復する、盗賊は当たれば止める、エルフは遠くから雷、魔法使いは……近接に出なくていい杖。

【レンツ】待った。**魔法使いの「射程が変わる杖」はリーダー時の反映が未検証**だ。仲間側は `if (w.range) ally.weaponRange = w.range`（16157）で確実に効くが、リーダーの `playerStats.weaponRange` は職業定義から来ていて、武器の range を拾う経路（13694付近）が全パスを通っているか確かめていない。**Phase 2 で個別に検証する項目に落とす**。Phase 1 が盗賊なら近接だから影響しない。

【ミサキ】……のみます。じゃあ Phase 1 は盗賊で。

【ガイウス】ひとつテーマを与えたい。第1弾が「対竜」、第2弾が「対アンデッド」なら、第3弾は**敵から逆算しない**——これは俺が候補①に感じていた薄さの答えでもある。**「解放前の職人たちが遺した、その職のための一点物」**。復興期のフランで、失われた技で打たれた品が各地に散っている。これなら5本がバラバラの属性でも、一本の筋が通る。

【ノエル】テストの手順だけ確認させてください。dev shop に並ぶのは**リーダーの職業のプール**なんですよね？

【レンツ】そうだ。`shopPool()` が `partyComposition[0]` の職を引く。だから**盗賊の魔剣を1Gで買うには、盗賊をリーダーにして酒場を開く**必要がある。手順としては自然だが、ドライバには明記しておく。

【ノエル】了解。それと**負のコントロール**を必ず入れてください。「戦士で tier 4 を所持している状態で、盗賊に切り替えたら未所持であること」。これが通らなければ今回の目的の半分は達成できていない。

## 第3ラウンド：合意

【レンツ】フェーズはこうだ。**Phase 1 = 基盤＋盗賊1本**（漏れを塞ぐのが主目的）。**Phase 2 = 残り4職**（データ追加が主体、魔法使いだけ射程の検証が付く）。**Phase 3 = 任意**で、候補③の「小さな代償」を後付けする。Phase 1 と 2 の間で窓を更新できる粒度にしてある。

【ガイウス】異論なし。

【ミサキ】新規アセットはゼロ。素材調達の段取りも不要です。

【ノエル】iOS 実機は「盗賊リーダーで magic drop → 装備 → 効果が乗る」の1本道だけ確認します。性能影響は無い見込み。

# 開発計画書：五職の一点物 — クラス別マジック武器

## 概要

非戦士リーダーが gated（マジックアイテム）武器を1本も引けない不均衡を解消する。5職の武器プール末尾に gated 武器を1本ずつ追加し、抽選プールをリーダーの職業に一般化する。同時に、gated 装備の所持を「共有ティア」から「名前キー」へ移し、1回のドロップが5職ぶんの魔剣を解錠してしまう漏れを塞ぐ。テーマは**「解放前の職人が遺した、その職のための一点物」**。

## 実装スコープ（Phase 1 で完結）

**作るもの**

## リスクと対策

| リスク | 対策 |
|---|---|
| **`opts.extraBonus` を使うと非装備者に乗る** | 使わない。`checkScore` の個人ボーナス方式で「持ち主だけ・代表選出にも反映」 |
| **共有ティアの漏れが防具にも波及** | 名前キー化を weapon/armor/shield 共通で。ドライバに**負のコントロール**（盗賊で tier3 所持 → ドワーフで未所持）必須 |
| `dfWeaponTierGated` が weapons 専用 | 種別対応へ一般化。armors/shields も gated 判定できるように |
| **効果が地味で伝わらない** | 判定パネルの内訳表示（既存の「技+N」）が答え。**ここが出ることをドライバで assert** |
| Phase 3 のセーヴ挿し忘れ | grep 全数監査＋経路別 assert。`hasFreeAction` の前例に倣う |
| 装備しても代表に選ばれない | `checkScore` 方式なら選出が変わる。**「装備で代表が交代すること」を assert** |
