# 開発会議〔2026-06-25〕：スマホUIの見やすさ改善（第2弾・縦持ち）

**議題**: モバイル(iOS Safari)の見やすさ改善。ユーザー実機スクショ2枚で報告 =
(1) 縦持ち・銀の鹿亭でシナリオカード/依頼人ポートレイト/怪しい扉が重なって判読困難、
(2) 戦闘画面のコンパクトPTステータスでHPバーが見えない・「あなた」が見切れる。

調査(Explore×3＋直読)で確定した根本原因:
- HPバー不可視 = `.chipBar` 8px ＋ 緑ゲージ×茶背景#5a4323 の低コントラスト。pct計算バグではない。
- 「あなた」見切れ = `.hpChip{min-width:64px}`×(4人+撤退) が画面幅超過 → `#hpMiniBar{overflow-x:auto}` で末尾スクロール外。
- 巨大「探索フェーズ」が上半分を覆う = `#phaseIndicator` のモバイル@media が `font-size:56px` で2行折返し肥大。
- `[AUTO PILOT]…でインベントリ` 見切れ = `#info{right:8px}`＋safe-area非対応。スマホにキーボード無く [I] ヒント自体が不適。
- tavern重なり = `#tableArea`/`.table`(200px幅)/%絶対座標/`#plazaDoor` が全て横長前提・@media皆無。
