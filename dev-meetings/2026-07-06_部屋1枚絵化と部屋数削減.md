# 開発会議〔第1段：候補出し〕：マジックアイテム路線の次、毛色の違う実装テーマ

**議題**: 対竜・対アンデッド装備が一段落。アイテム以外の方向（職業・シナリオ・演出・QoL）も含め、次に作る一手を発散して1つ選ぶ。
**ゴール**: 毛色の違う候補を並べて、進む方向を1つ決める。

---

## 第1ラウンド：ブレスト（各専門領域から1つずつ）

【ガイウス】本筋もポストゲームも締めた今、足りないのは「もう一度潜る理由」。ポドルプラザを入口にした反復討伐＝アリーナ／賞金首モードを推す。クエスト生成エンジンとスポーン定義を回すだけで冒険体験を無限供給できる。

【レンツ】中身の濃さで攻めたい。状態異常システム（毒・出血・炎上・スタン）が薄い。ダメージ計算パイプラインに継続効果レイヤーを差せば、既存の攻撃・呪文・敵能力が一気に立体化。実装は相乗り、新規最小。

【ミサキ】部屋がまだタイルの床と壁。バックログに寝てる「部屋を1枚絵で描く」パイロットを起こしたい。1部屋だけ painterly にしてフォグと松明を絵の上に乗せる。世界の格が一段上がる。

【ノエル】オートバトルは「今なにが起きてるか」がiPhone片手で追いにくい瞬間がある。ダメージ表示・戦況ログ・撤退の判断材料——戦況可読性とリザルト画面を整える。地味だが底上げは一番効く。

## 第2ラウンド：絞り込み

【ノエル】1枚絵は1部屋だけ絵が浮くと逆に安っぽく見えるリスク。全面展開の判断ができる粒度に絞るなら賛成。
【レンツ】工数順は 状態異常 ≦ アリーナ < 戦況可読性の一部 < 部屋絵。状態異常は素材ゼロ・コードのみ。
【ガイウス】状態異常はD&Dらしさを最も増す。アリーナと相性◎（周回に駆け引きが乗ると作業感が減る）。
【ミサキ】部屋絵は今回は温存でいいが次点に。状態異常やるなら演出はこっちで持つ。

# 開発会議〔第2段：実装計画〕：部屋を1枚絵化＋部屋数削減

**選択案**: ダンジョンの部屋を painterly な1枚絵で描く方式をパイロット導入。同時に部屋数を現在の4→3へ削減。

## 調査で判明した核心（Explore 2体）
- 部屋絵インフラ `ROOM_PAINTINGS_DEF`（index.html:2872-2951）は v7.4 で**データだけ空にされ無効化**。機構（フェザー/タイルスキップ/フォグ上乗せ/Pass 1b.5 drawImage）は完全に生きている。撤去理由＝旧6部屋比率PNGが新4部屋幾何で歪んだ。
- → **幾何を最終形にロック→絵は後**が鉄則（逆順はv7.4の轍）。「削減が先、絵が後」。
- 部屋は全6シナリオ共通の単一 `ROOMS`（現在4部屋）。減らすには `ROOMS`/`CORRIDORS`＋幾何依存7箇所＋tavern.html の `ROOM_SLOTS`/`buildSpawns`（3部屋前提の `/3`）を同時修正。
- フォグ/光は別canvas(z9)なので絵(z0)の上に自動で乗る。合成順の追加コード不要。

## 議論の要点（対話）
- 【レンツ】順序を握る。削減→絵。既存 `ROOM_PAINTINGS_DEF` を埋めるだけで新規機構ゼロ。
- 【ガイウス】4部屋→中だるみの「ボス直前」を削り、導入→山場→ボスの三幕（3部屋）が構造的にきれい。2部屋は短すぎ。
- 【ミサキ】部屋を減らすほど全面展開の枚数（4部屋24枚→3部屋18枚）が減る。パイロットは最初に触れる goblin-mine 山場。
- 【ノエル】フォグはタイル解像度のまま乗る→「絵単体で綺麗」でなく「フォグ・松明込みで綺麗」を合格ラインに。
- 【ミサキ】1枚絵の部屋は床グリッド線を消す（Pass 1.2 で isPaintedAndLoaded タイルをスキップ）。
- 【レンツ】部屋削減で checkDungeonClear の `ROOMS.length-2`、tavern の `/3`、SCENERY_RECIPES キー、全6 spawns の col が連動→一括修正後にヘッドレス検証。
- 【ガイウス】1周短縮で `applyRoomClearHeal` 機会が減り体感難度が上がる方向→実機で微調整（机上で決め打たない）。

## 決定
- **部屋数=3（導入→山場→ボス）** / **パイロット=goblin-mine 山場**
- **フェーズA（幾何削減・コードのみ）を緑にしてからフェーズB（絵）**
