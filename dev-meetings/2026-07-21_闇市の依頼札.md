# 第1段：発散会議(候補出し)

## 会議前に共有された実コードの事実

- 現行の酒場掲示板は**左下の小パネル**(`#questBoard`)。`.qcard` は罫線の箱で、
  本文(`_sentence`)は**選択した1件しか見えない**。
- 酒場のメイン画面は**6卓のテーブル**に固定シナリオの依頼人が座る形(`renderTables`
  → クリックで `openDialog(sc)` = 「クエ主がしゃべる画面」)。
  **依頼人立ち絵 `assets/client_<id>.png` は6枚とも既に存在**。
- 生成側の素材点数: 依頼人 **12種**(`clients` 8 + `clientsHigh` 4、**ただの文字列**で
  ポートレイト欄がない) / モンスターfamily **11種** / places 10種。
  闇市の `plazaClients`(3人)だけは既に `portrait` フィールドを持つ。
- **報酬はフレーバー文字列**("金貨300枚" 等)。モックの「120 ゴールド」に相当する
  確定報酬はゲームに存在しない(ゴールドは道中ドロップ `earnedGold`)。
  XPだけ実数(`clearXp` 300/600/1000/1500)。

## 第1ラウンド：ブレスト

【ガイウス】モックの札は3列×2行で**6枚**。うちの**固定シナリオも6本**だ。生成クエストは
無限に湧くから文字を焼き込めんが、**固定6本なら焼き込める**。あの札を本編6シナリオの
顔にするのが一番D&Dらしい。

【レンツ】逆を言う。既存の掲示板はデータ駆動でもう動いてる。ただ `clients` は**ただの
文字列配列**だ。挿絵を出すなら `{name, portrait}` へ昇格が要る。闇市の `plazaClients` は
既にその形。依頼人12 + family 11 = **挿絵23枚**が素材予算の実数。

【ミサキ】あのモックがいいのは羊皮紙でも枠でもなく、**ペン画の怪物が「その依頼の顔」に
なってる**こと。FFの線画はハッチングだけで陰影を作るから、羊皮紙に multiply で乗せると
インクが紙に染みたように見える。

【ノエル】iPhone縦持ちだと**1枚あたり横120px**。依頼文4行＋説明2行＋報酬行が入るわけない。
それと根本的な話、**今の掲示板は本文が選択した1件しか見えない**。モックの本当の価値は
羊皮紙じゃなく「**並べて読み比べられる**」ことでは?

## 第2ラウンド：絞り込み

【ノエル】挿絵23枚を先に作って後から「読めない」となったら23枚が無駄になる。順番の問題。

【レンツ】ただノエル案は画像ゼロで、慶彦さんの「絵を作らせたい」に答えてない。折衷案:
**羊皮紙テクスチャと枠飾りだけ先に作る**。無限生成と無関係だから何件湧いても使い回せる。

【ミサキ】羊皮紙だけの札は**たいてい「ダサい茶色い箱」になる**。せめて依頼人の顔12枚は欲しい。

【ノエル】ガイウスさんの6枚案が一番リスクが低い。6枚固定なら**文字ごと焼き込んだ完成品**が
使えて、モックとピクセル単位で同じ絵が出せる。

【レンツ】だが酒場は**6卓に依頼人が座ってる絵**だ。札の壁で置き換えたら `client_*.png` 6枚と
円卓レイアウトが死ぬ。併存させると**入口が2つ**になって迷子になる。

【ミサキ】じゃあ札は**卓の上に置く**。テーブルはそのまま、上に依頼書が伏せてある。

【レンツ】で、**報酬の数値**は誰も触れないのか。モックには「120 ゴールド」とある。
**うちに確定ゴールド報酬という概念はない**。書くならルール変更が要る。

【ノエル】書かないか、ルールを足すかの二択です。

## 候補一覧

| # | 名前 | ジャンル | 規模 | 流用度 | 推し / 慎重 |
|---|---|---|---|---|---|
| ① | 札の化粧直し(羊皮紙だけ) | 演出強化(最小) | 小 | 高 | ノエル / ミサキ |
| ② | 卓の上の依頼札(固定6シナリオを札に) | 体験設計 | 中 | 高 | ガイウス / レンツ |
| ③ | 挿絵パイプライン(生成クエストに顔と怪物) | システム+美術 | 大 | 中 | ミサキ / ノエル |
| ④ | 札を1枚ずつめくる(縦持ち最適化) | UX | 小〜中 | 高 | ノエル / ガイウス |

### 未決の争点
1. **報酬の数値問題** — モックの「120 ゴールド」に相当するものがゲームに無い。
2. **固定シナリオ vs 生成クエスト** — どちらを札にするか、両方なら順番。

## 🎯 慶彦さんの決定

> 「酒場は、そのままで行こう。ポドルプラザの、掲示板を選ぶと、このカードの中から
> 選べるようになる。。でどうかな?」

→ **候補外の第5案。酒場は不変、闇市の掲示板を札のカード一覧にする。**

# 第2段：収束会議(実装計画)

**選択案**: 酒場は現状維持。闇市ポドルプラザの「闇クエスト掲示板」を、羊皮紙の依頼札を
並べて1枚選ぶ形にする。受注後は既存の受注ナレ(依頼主がしゃべる画面)へ。

## 第1ラウンド：設計

【レンツ】この選択は**技術的に一番安い**。掲示板の描画は `renderPlazaBoard()`
(tavern.html:5141) 一関数で `board.innerHTML` を組み立ててるだけ。選択ロジック
(`plazaSelectedIdx`)も出発配線(`onPlazaDepart` → `buildPlazaSynthetic` → `openPrep`)も
**一切触らない**。しかも**掲示板は tavern.html にしか無い** = **二重同期コストゼロ**。
酒場側を選んでいたらこうはいかなかった。

【ミサキ】**闇市の依頼主は既に立ち絵を持ってる**。`plazaClients` の3人(仲買ゴブリンのジグ /
傭兵隊長ホブゴブリンのグラーグ / 沼の商人リザードのシャザ)全員 `portrait` があり
`assets/plaza_client_*.png` が実在。**モックの「依頼人の顔」枠は新規1枚も描かずに埋まる。**

【レンツ】それと前回ノエルが刺した**報酬の数値問題は闇市では起きない**。闇クエストには
`fangReward`(牙貨)という**実数**が既にある。

【ノエル】**枚数の話をします**。`rollPlazaQuests` の通常枠は**1〜2件**(「レア感」とコメント)。
今は稀少枠(単眼の暴君・隊商護衛)が**テスト値で100%**なので4枚並んで見えるが、
**商用値0.08に戻したら1〜2枚しか並ばない**。1枚では「この中から選んでくれ」が成立しない。

## 第2ラウンド：肉付けと衝突

【レンツ】件数を増やすと闇市が「もう一つの掲示板」になり酒場との差が消える。レア感が売りだ。

【ミサキ】でも1枚しか出ない札の壁は見た目として**失敗**。空きを何で埋めるの?

【ノエル】逆に、**空き枠を「剥がされた跡」にする**。破れた札の切れ端、錆びた釘、血の染み。
「ここに貼ってあった依頼はもう誰かが持っていった」。1枚でも画面が成立し、レア感は
むしろ**強調される**。

【ガイウス】空白は物語だ。ただし**最低2枚**は保証しろ。通常枠を1〜2 → 2〜3へ。

【レンツ】2〜3なら飲める。

【ミサキ】闇市のテーマ色は**燐光グリーン**(`--parchment-gold: #45e0a8`)。モック通りの
セピア羊皮紙とは**喧嘩する**。

【ガイウス】喧嘩させろ。闇市の札が小綺麗な羊皮紙のほうがおかしい。**血と煤で汚れた札**に
して、燐光は選択枠と牙貨の数字にだけ残せ。

【ミサキ】悔しいが正しい。ただし将来の酒場展開のため**同じ構造で色だけ変わる**設計に。
紙の色と汚れ強度をCSS変数で持つ。

【ノエル】`plaza_client_*.png` は**1枚500KB**。3枚で1.5MB、しかもこの画面には
`plaza_bg.png` が**2.1MB**ある。**札用サムネを別に作る**か遅延読込にしないとカクつく。

【レンツ】札用は幅200pxで十分。既存PNGから縮小するだけ、新規生成不要。

【ガイウス】受注後の流れは?

【レンツ】**既に動いてる**。`playQuestAcceptNarration` (tavern.html:4761) で闇市クエストは
`clientPortrait` を持つため**受注ナレに依頼主の立ち絵が既に出る**。触る必要なし。

## 第3ラウンド：合意

STEP1 = 新規画像1枚(汚し羊皮紙)だけで札にする。顔は既存3枚の縮小版 → iPhoneで確認。
STEP2 = モンスター挿絵11枚(FF風ペン画バッチ)。
STEP3 = 空き枠演出(剥がされた跡) + 通常枠2〜3への調整。

# 📋 開発計画書：闇市ポドルプラザの依頼札

## 概要
闇市の「闇クエスト掲示板」を、羊皮紙の依頼札を並べて1枚選ぶ形に作り替える。
機能・データ・出発配線は既存のまま、`renderPlazaBoard()` が吐く HTML と CSS だけを
差し替える。酒場は現状維持。

## 実装スコープ(STEP1で完結)
- **作る**: 札のマークアップ/CSS、汚し羊皮紙テクスチャ1枚、依頼主サムネ3枚(既存PNG縮小)
- **作らない**: モンスター挿絵、空き枠演出、件数調整、酒場への展開(すべて後段)

## 流用する既存システム
- `renderPlazaBoard()` (tavern.html:5141) — innerHTML の組み立てだけ差し替え。選択/出発は不変
- `plazaClients` の `portrait` — 依頼主の顔はデータにも画像にも既に存在(3人)
- `q.fangReward` — 牙貨は**実数**。モックの数値報酬に相当する枠が闇市にだけ既にある
- `playQuestAcceptNarration` (tavern.html:4761) — 受注後の「依頼主がしゃべる画面」は配線済み
- `window.__plazaTV` — ヘッドレス検証ブリッジ(`rollPlazaQuests` / `getPlazaQuests`)

## 新規実装
- 札の DOM 構造(リボン見出し / 依頼主 / 依頼文 / 罫線 / 討伐対象 / 報酬行)
- CSS変数化した紙テーマ(色・汚れ強度) ← 将来の酒場展開のため
- 依頼文(`_sentence`)を**全札に常時表示**(現在は選択中の1件しか読めない)

## 素材調達
- **自動生成分**: 汚し羊皮紙テクスチャ 1枚 — `tools/chatgpt_generate.py`
- **ローカル生成分**: `assets/plaza_client_{goblin,hobgoblin,lizardfolk}.png` の縮小サムネ3枚
  (Pillow、DL不要)
- **NG確認**: 外部素材サイト・アカウント登録は一切不要
- **慶彦さんの作業**: 生成された羊皮紙を見て採否を決めるだけ

## iOS Safari 検証項目
- 闇市を開いた瞬間のカクつき(`plaza_bg.png` 2.1MB との同時読込)
- 札の同時デコード枚数とメモリ
- 縦持ちで依頼文が読めるか
- タップ領域(札全体がタップ可能か)
- 既存の折りたたみパネル(`aria-expanded`)との整合

## 実装ステップ
1. 汚し羊皮紙テクスチャを生成 → 採否判断
2. 依頼主サムネ3枚を既存PNGから縮小書き出し
3. `renderPlazaBoard()` を札構造へ + CSS 追加
4. ヘッドレス検証(札枚数・選択の付替え・出発配線が壊れていないこと)
5. iPhone実機で目視 → 件数と空き枠演出の要否を判断

## リスクと対策
- 札が1〜2枚しか並ばない → STEP3で「剥がされた跡」演出 + 通常枠2〜3へ
- 燐光グリーンと羊皮紙セピアの衝突 → 汚し版の紙にし、燐光は選択枠/牙貨だけに残す
- 画像重量で入場がカクつく → サムネ縮小 + `loading="lazy"`
- ⚠️ **稀少枠が現在テスト値100%**(`RARE_SOVEREIGN_CHANCE` / `RARE_CARAVAN_CHANCE` = 1.0、
  tavern.html:5052)→ 商用前に 0.08 へ戻す。**戻した時の見え方をSTEP1で必ず確認**

## 二重ファイル同期
**なし**。掲示板は tavern.html にのみ存在。index.html は無改造。
