## 第1ラウンド：ブレスト（各専門領域から方向性）

【ガイウス】いいか、これはD&Dなんだ。わしが自分の目で見て一番心を動かされたのは `index.html:14232-14243` の `templeBlessing` だ。神殿でカエルムの霊を弔うと選んだ者だけが恒久の恐怖免疫を localStorage に刻み、それが二本先のシナリオ、ファラクサスの `fearDC: 18` を無力化する。たった3行。だがこれは Gold Box が『Curse of the Azure Bonds』でやったこと、すなわち「前の卓での選択が次の卓の盤面を書き換える」そのものだ。そして——**これがこの作品で唯一の、シナリオを跨ぐ帰結なんだよ**。

### 会議で起きた対立（折衷で着地）

1. **「賢いAI」は片手縦持ちで観測できるのか**（レンツ × ノエル）
   - レンツ「一様ランダム抽選は構造的欠陥。直せば全戦闘・全シナリオに効く。`pickLeaderAction` 1関数、描画コストもゼロ」 ⇔ ノエル「戦闘ログは iPhone で可視3〜4行、しかも `scrollTop` 無条件固定で遡れない。判断根拠が見えない以上、**賢くなったこと自体が観測不能**」
   - → **「選択理由の1行表示」を実装の必須要件（オプションではなく完了条件）に格上げ**した上で採用。さらに候補④のログ改善を同一リリースに束ね、賢さが読める土台を先に敷く。AoE と selfStun のスコアリングは初回スコープ外。

2. **ボス部屋か、山場か ── 絵をどこに置くか**（美咲 × ノエル/ガイウス）
   - 美咲「クライマックスのボス部屋が6シナリオ全部でタイル描画のままなのは逆進行。絵を注ぐならボス部屋6枚だ」 ⇔ ノエル/ガイウス「ボス部屋は 22x18=アスペクト1.222 で既存4枚の1.429と**不一致**、遅延ロードも新規実装、常駐メモリは img+feather canvas の二重で 12.6MB/枚。しかも iPhone縦持ちで一度に見えるのは**幅の18%**、そもそも構図が見えない」
   - → ボス部屋6枚は却下。同一アスペクトで本当に「テーブルに2行」で済む**山場(ROOMS[1])の未適用2本**（undead-temple / dragon-lair）を先に埋める。ラスボス面が全編タイルという最悪の状態はこれで解消。絵で足りない「終着点の格」はラスボス固有台詞と撃破ボイスで補う。

3. **守護者を「味方化」できるか**（ガイウス × レンツ）
   - ガイウス「酒場が『振る舞いで決まる』と言っている以上、味方化ルートが無いと嘘のままだ」 ⇔ レンツ「`faction` は beast/非beast の**二値**でしかなく、コード内に『新しい値は無効』という警告が2箇所（6425-6426 / 18880）。味方化には対象選定・エンカ検出・クリア判定の4面改修が要るし、撃破XPが全額プレイヤーに入るのでバランスも溶ける」
   - → **味方化を丸ごと捨て、残すのは「選択と声」だけ**。①静かに引き返す ②起動させる ③刻印を読み上げる（判定）。`faction` には一切触らない。副産物として `physicalImmunity` 持ちの守護者による**潜在ソフトロックが塞がる**。

4. **「新しい楽しさゼロ」の大掃除を今回も入れるか**（ノエル × 美咲/ガイウス）
   - ノエル「『出発する』の直下に全6シナリオ自動巡回ボタンがあり、セーブのリセット導線が存在しない。商用配布を掲げる作品として**仕様以前の出荷事故**」 ⇔ 美咲/ガイウス「前回も『新規ゼロ』で落ちた案。引き算だけの回にしたくない」
   - → 3案（デバッグ隔離 / データ消去 / ログのsticky-bottom+行詰め）に絞り、実測で体感が変わらないと判明した戦闘速度はスコープ外。**単独の目玉ではなく候補③の観測基盤として同梱**する形で採用。

## 候補③：英雄の眼 ── リーダーの一様ランダム抽選をやめ、選んだ理由を1行出す
- **ジャンル**: 戦闘システム

- **ざっくり規模**: 中 ／ **既存流用度**: 中 ／ **iOS Safari**: クリア ／ **体感インパクト**: 3/5
- **流用**: `pick`(23176・差し替え対象) / 候補フィルタ(23150-23172・無改造) / `evaluateThreat`(21304-21320) — **ただし書き換えない**（ボス判定が `isBoss` ではなく `maxSummons>0` になっており、ハイドラ・単眼の暴君ら6体でボス加点が乗らない。`pickLeaderAction` 内で `def.isBoss || def.eyeStalks` をローカル計算＝14963 と同じ式にする）/ `getRange`+`tileChebyshev`(23179-23193) / `updateInfo`(10047) = 理由1行の出力口

## 候補④：出荷前の大掃除 ── デバッグ隔離・データ消去・読めるログ
- **ジャンル**: QoL/UX（片手縦持ちの事故潰し）
- **狙い**: 「出発する」の真上と真下にある**無ゲートのデバッグ5ボタン**、**リセット不能な20キーのセーブ**、**遡り読み不能な戦闘ログ** ── 電車で片手で遊ぶ人が確実に踏む地雷を3つ潰す
- **ざっくり規模**: 小 ／ **既存流用度**: 高 ／ **iOS Safari**: クリア ／ **体感インパクト**: 3/5
- **流用**: `tavern.html:1578-1590` デバッグ5ボタン / `unlockall`(2367)・`givemagic`(2880) の URLパラメータ前例 / `el.scrollTop = el.scrollHeight`(9799・ここを条件化) / `COMBAT_LOG_MAX=14`(9778) / line-height:1.5(2044-2045) / ::before「戦闘ログ」見出し(2050-2059) / `SETTINGS_KEY`(audio.js:12・リセット除外対象)
- **新規**: ①デバッグ5ボタンを `?dev=1` ゲートへ（復帰4経路でフラグが落ちるので sessionStorage に焼く） ②設定に「冒険の記録を消す」（二重確認 + `dragonfighters.*` 列挙削除、`dragonfighters.settings` は除外） ③`appendLog` の scrollTop 固定を「最下部から24px以内なら追従」に条件化 ④ui-compact 時のみ line-height 1.5→1.3 / padding 10→6 / 見出し非表示で**可視行を約1.7倍**（**枠高は不変=`computeUiMetrics` と屋外フィールド幾何に触れない**） ⑤敗北文言9箇所を世界観語へ。**戦闘速度は今回スコープ外**

## リスクと対策（実測に基づくもののみ）
- **理由行が④で稼いだ可視行を食い潰す**（3人が独立に指摘。1ラウンドは最低8行出る）→ 三重に緩和: ①`pick !== 'normal'` 限定 ②閾値『勝者の weight/base >= 1.5』 ③`COMBAT_LOG_MAX` 14→18。検証で『1ラウンドあたり理由行が最大1行』を assert

- **CSS ゲートがインラインスタイルに負けてデバッグ5ボタンが露出したまま出荷される** → インラインの `display:flex;` を**除去**（`!important` で殴る代替は将来の優先度戦争を招くので採らない）。assert は『存在しないこと』ではなく『`getComputedStyle().display === "none"` かつ rect.height === 0』で**defect を測る**

- **`期待値 × (1-対象HP比)` の素の乗算だと、新品の敵に対して重みが厳密に 0 になり開幕が構造的にバフ偏重になる**（敵生成は `hp===maxHp`）。しかも targetIdx は単一値に確定済みなので HP項は**完全に約分される** → HP項を `0.6 + 0.8*(1-tgtRatio)` の帯（0.6〜1.4、決して0にならない）に写像し、候補ごとに真偽が変わる**撃破判定**（期待値 >= 残HP なら ×1.8）を足す
- **`Math.pow(w, 1/T)` と末尾の丸め保険 return の組み合わせが、非有限値を静かに「常に最後の候補」へ変換する**（T=0 なら NaN → `NaN<0` は常に false → 全ループ空振りで保険 return。`total <= 0` ガードは NaN を検出しない）→ T をクランプ / ガードを否定形に / 非有限を `console.warn` / **『1戦闘を通して当該 warn が0件』を assert**
